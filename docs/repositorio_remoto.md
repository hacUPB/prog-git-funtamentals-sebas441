Para crear un repositorio remoto en GitHub y sincronizarlo con un repositorio local,
 primero se crea el repositorio en GitHub asignándole un nombre y eligiendo su visibilidad.
Luego se verifica que el proyecto local ya esté bajo control de Git y tenga al menos un registro de cambios.
Después se conecta el repositorio local con el repositorio remoto usando la dirección
proporcionada por GitHub, lo que permite que ambos se comuniquen. Finalmente,
se envían los archivos y el historial de cambios desde el repositorio local al remoto,
quedando sincronizados para poder seguir trabajando y actualizarlos en el futuro.
